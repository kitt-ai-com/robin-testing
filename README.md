## 2. 검증 범위

셀러 ↔ 관리자 양방향 동기화와 엑셀 업로드 / 템플릿 / AI 매핑 동작을 점검한다.

- 셀러가 등록한 **상품 매칭(별칭)** 이 관리자 화면에 동일하게 보이는가
- 셀러의 **엑셀 템플릿** 이 관리자 화면에 동일하게 보이는가
- 관리자가 셀러를 선택해 **건별 주문 등록** 시 해당 셀러 화면에 정상 반영되는가
- 관리자 셀러 상세 모달에 셀러의 매칭 내역이 노출되는가
- AI 엑셀 업로드의 파일 포맷 제약 확인
- AI 매핑(컬럼명/상품명이 표준과 달라도 자동 분류) 정확도 — *별도 엑셀 파일 필요*

## 3. 상세 결과

### 3-1. 셀러 → 관리자 매칭 동기화 ✅

- 셀러 화면(`/seller/aliases`) 등록 3건:
  - `QA-활전복-TEST`
  - `상품옵션:비품 급냉 홍게 4kg/7~13미`
  - `121212`
- 관리자 화면(`/admin/sellers/{id}/aliases`) 에서 동일 3건 확인

### 3-2. 셀러 → 관리자 엑셀 템플릿 동기화 ✅

- 셀러 화면(`/seller/excel-templates`) 0건 → 관리자 화면도 0건
- 저장된 템플릿 없음 상태가 동일하게 노출됨

### 3-3. 관리자 → 셀러 주문 생성 동기화 ✅

관리자 화면에서 **건별 주문 등록**으로 생성:

| 항목 | 값 |
|---|---|
| 셀러 | 홍길수산 |
| 상품 | QA-활전복-TEST (50,000원) |
| 수령자 | 테스트수령자 / 010-1234-5678 |
| 주소 | 서울특별시 강남구 테헤란로 1 (06234) |
| 외부주문번호 | QA-TEST-260518-01 |
| 메모 | QA 테스트 주문 - 관리자에서 생성 |

결과: 주문번호 **ORD-260518-5915** 생성 → 셀러 로그인 후 `/seller/orders`에서 정상 노출, 대시보드 매출 8,600원 → 58,600원으로 증가.

### 3-4. 관리자 모달 매칭 노출 ✅

관리자 셀러 상세 모달에서 셀러의 매칭 3건이 그대로 노출됨.

### 3-5. PDF 업로드 ⛔

- 파일 입력 속성: `accept=".xlsx,.xls"`
- AI 엑셀 업로드 모달도 동일 제약
- **결론**: PDF 지원 안 됨

### 3-6. AI 매핑 정확도 🔜

공유 시트(`주식회사왓구나WGUNA_..._20260506111323774`, 14개 컬럼, 2개 데이터 행)로 점검 예정. 항목:

- 컬럼명이 표준과 달라도 AI가 표준 컬럼에 자동 매핑하는가
- 매칭이 없는 상품(예: `생물 호래기 500g`)에 대해 AI 처리가 적절한가
- AI 업로드 시 템플릿이 자동 생성되는가
- 템플릿 저장/수정/삭제/복제 동작
